Remote IPython kernel control via HTTP & Pi extension

Python TypeScript FastAPI Pi

Overview

ipyforge-kernel is a two-part system for controlling an IPython kernel from an LLM agent (Pi):

🚀 FastAPI Server

Thin HTTP/JSON bridge wrapping jupyter_client.BlockingKernelClient. No external dependencies beyond jupyter-client, ipykernel, and fastapi.

🧩 Pi Extension

TypeScript extension registering 6 custom tools that call the server via fetch(). Enables an LLM to execute Python code, evaluate expressions, and inspect kernel state.

Setup

  1. Install dependencies 
    uv sync
  2. Create the IPython profile (one time) 
    uv run ipython profile create agno
  3. Start a kernel (for remote access) 
    uv run ipython kernel --profile=agno --ip=0.0.0.0 -f /tmp/remote-kernel.json

    This writes a connection file to /tmp/remote-kernel.json. Copy this file to where the server will run and update cfg.json.

Server

Each request establishes a fresh connection to the kernel — reads kernel.json, creates a BlockingKernelClient, verifies with kernel_info(), performs the operation, then disconnects.

uv run python server/main.py

Reads configuration from cfg.json in the current working directory:

{
  "port": 9123,
  "kernel_connection_file": "/tmp/remote-kernel.json",
  "max_output_chars": 20000,
  "default_timeout_s": 30
}

Connecting to a Kernel

Once a kernel is running and its kernel.json is accessible, connect using the console:

# On the same machine:
uv run jupyter console --existing /tmp/remote-kernel.json

# From another machine — edit the "ip" field in kernel.json first:
uv run jupyter console --existing /path/to/copied/kernel.json

Or connect via the Pi extension's kernel_connect tool.

Pi Extension

Copy extension/index.ts to Pi's auto-discovery path:

# Global install:
python install-extension.py

# Per-project install:
python install-extension.py --local
python install-extension.py -p /path/to/project

Then run /reload in Pi.

API Reference

GET /health

Health check.

{ "status": "ok" }

POST /kernel/connect

Verify and optionally set the default kernel connection file.

ParameterTypeDescription
connection_filestring?Path to kernel.json
set_defaultbool?Use as default (default: true)

POST /kernel/run-code

Execute Python code and return captured output. Output is truncated per max_output_chars in cfg.json.

ParameterTypeDescription
codestringPython code to execute
timeout_snumber?Timeout in seconds (default: 30)
{
  "output": "4",
  "truncated": false
}

POST /kernel/eval-expr

Evaluate a Python expression using user_expressions. Non-history-polluting.

ParameterTypeDescription
exprstringPython expression (e.g., len(data))
timeout_snumber?Timeout in seconds (default: 30)
{ "result": "42" }

POST /kernel/interrupt

Send SIGINT to the kernel (aborts a stuck execution).

{ "interrupted": true }

POST /kernel/get-output

Return a slice of the last full output from /kernel/run-code.

ParameterTypeDescription
startnumber?Character offset (default: 0)
limitnumber?Chars to return (default: 4000)
{
  "output": "...",
  "start": 0,
  "end": 4000,
  "total": 12345
}

GET /kernel/status

Show connection and server state.

{
  "connected": true,
  "connection_file": "/tmp/remote-kernel.json",
  "port": 9123
}

Pi Extension Tools

ToolDescriptionParameters
kernel_connect Connect to a kernel via kernel.json connection_file?, set_default?
kernel_run_python Execute Python code, return output code, timeout_s?
kernel_eval_expr Evaluate a Python expression expr, timeout_s?
kernel_interrupt Interrupt the running kernel —
kernel_get_output Retrieve cached output slice start?, limit?
kernel_status Show server and connection state —
